# Product Guidelines

## 1. User Experience (UX) Principles
- **Simplicity First:** The interface should be intuitive and clutter-free, focusing entirely on the estimation process.
- **Real-time Responsiveness:** Actions such as joining a lobby, voting, and revealing cards must feel instantaneous to all participants.
- **Clear Feedback:** Users should receive immediate visual confirmation when they select a card or when the round state changes.

## 2. Design & Branding
- **Professional yet Engaging:** Use a clean, modern aesthetic with ample whitespace. The card design should be distinct and easy to read.
- **Accessibility:** Ensure high contrast for text and interactive elements. Support keyboard navigation and screen readers for core functionalities.
- **Consistent Typography:** Use a readable, standard web font (e.g., Roboto, Inter) across the application.

## 3. Prose & Communication
- **Tone:** Professional, clear, and encouraging. Avoid overly technical jargon where simple terms suffice.
- **Error Messages:** Provide actionable and polite error messages (e.g., "Unable to join the lobby. Please check the link and try again.").

## 4. Performance Expectations
- **Low Latency:** WebSocket communication should be optimized to minimize delays between a user voting and the lobby updating.
- **Mobile Friendly:** The application must be fully responsive, allowing users to participate comfortably from smartphones and tablets.
